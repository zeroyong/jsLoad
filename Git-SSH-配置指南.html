🔐 Git HTTPS → SSH 切换

解决 RPC failed; curl 56 OpenSSL SSL_read: Connection was reset, errno 10054

❌ 问题现象

使用 git push 通过 HTTPS 协议推送到 GitHub 时失败：

error: RPC failed; curl 56 OpenSSL SSL_read: Connection was reset, errno 10054
fatal: The remote end hung up unexpectedly
fatal: The remote end hung up unexpectedly

⚡ 为什么

原因说明
🌐 国内网络环境GitHub 的 HTTPS 连接常被 GFW 干扰，导致 SSL 连接中途重置
📦 推送数据量大仓库含图片/二进制文件时，长时间的 HTTPS 流式传输更容易被重置
🔌 SSL 协议层面OpenSSL 在 TLS 握手或传输过程中收到 RST 包，无法恢复
💡 核心思路：HTTPS 是被动防御（依赖 TLS+网络），SSH 是主动加密通道，更稳定，不会被中途拦截重置。

📋 解决步骤

1

git remote set-url切换远程仓库协议为 SSH

git remote set-url origin git@github.com:zeroyong/jsLoad.git
2

ssh -T测试 SSH 连接，发现权限问题

$ ssh -T git@github.com
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@ WARNING: UNPROTECTED PRIVATE KEY FILE! @
Permissions for 'C:\Users\zsm/.ssh/id_ed25519' are too open.
It is required that your private key files are NOT accessible by others.
This private key will be ignored.
Permission denied (publickey).

⚠ 原因 Windows 上 Git 的 OpenSSH 要求私钥文件只能被当前用户访问。由于权限继承，Everyone 组也有读取权限，SSH 出于安全拒绝使用该密钥。

3

icacls 修复私钥文件权限

禁用权限继承 + 移除 Everyone 组 + 仅保留当前用户完全控制：

icacls "$env:USERPROFILE\.ssh\id_ed25519" /inheritance:r /grant "${env:USERNAME}:(F)"
4

✅ 验证再次测试 SSH 连接

$ ssh -T git@github.com
Hi zeroyong! You've successfully authenticated, but GitHub does not provide shell access.
5

✅ 推送成功

$ git push origin master
c549ff5..be95f09 master -> master
