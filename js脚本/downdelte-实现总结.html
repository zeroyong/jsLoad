downdelte.user.js

目标网站：https://tongquet.com/book/*  ·  运行环境：手机端（触屏）

需求演变

v1.0删除回到顶部按钮

页面自带的 #back-to-top 按钮遮挡阅读，需要移除。

document.getElementById('back-to-top')?.remove()

问题：页面动态渲染，按钮加载时机不确定。

立即尝试删除，如果元素不存在则用 MutationObserver 监听 document.body，出现即删。

v1.1替换为字体调节按钮

不删除，把回到顶部按钮替换成调节 #reader-content p 字体大小的控制面板。

  • 固定定位右下角（同原按钮位置）
  • + / 当前字号 / - 垂直排列
  • 字号保存在 localStorage，刷新不丢失
  • 范围 12 ~ 36px

v1.2步长改为 1 + 自动隐藏半圆

每次 ±1px，5 秒无操作自动靠右收缩为半圆，避免遮挡阅读。

问题①：最初用 mouseenter / mouseleave 控制显隐，但这是手机端，没有 hover 事件。
全部改用 click 事件。点击半圆展开，点击 +/- 调整并重置倒计时。
问题②：用 translateX(calc(100% - 20px)) 做隐藏，漏出的直角边不像半圆，且 pointer-events: none 导致不可点击。
彻底分离展开/收起两套 DOM 结构：
  • 展开态：.fsc-panel — 显示 +/字号/-
  • 收起态：.fsc-knob — 44×44px 圆形 "Aa"
  • 用 right 属性过渡：展开 right: 20px，收起 right: -22px（圆心在视口右边缘，只露左半圆）
  • display: none/flex 切换面板/旋钮 + transition: right .35s ease
最终代码结构
downdelte.user.js
├── 元信息（@match, @grant 等）
├── STORAGE_KEY / 字号范围
├── getFontSize / setFontSize        ← localStorage
├── applyFontSize(size)              ← 遍历 p 设 fontSize
├── createControls()
│   ├── .fsc-panel（展开面板）        ← + / 字号 / -
│   ├── .fsc-knob（收起旋钮）         ← 44px 圆形 "Aa"
│   └── click 事件
├── collapsePanel()                  ← right → -22px
├── expandPanel()                    ← right → 20px
├── startAutoHide()                  ← 5s 倒计时
├── resetAutoHide()                  ← 展开 + 重置计时
├── btnStyle()
└── init()                           ← 删除原按钮，挂载控件
关键实现细节
  • 字号存储：localStorage('tongquet_font_size')，默认 18px
  • 半圆原理：44px 圆形 + right: -22px → 圆心在视口右边缘，露出左半圆
  • 纯 click 驱动：无需 hover，适合触屏
  • 事件冒泡：+/- 用 e.stopPropagation() 防止触发 wrap 的 click
  • 加载策略：检测 document.readyState，非 loading 则立即执行
  • 范围限制：Math.max(12, Math.min(36, size))
